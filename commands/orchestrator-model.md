오케스트레이터 모델 선택을 처리하세요. 인자: $ARGUMENTS

## 1. 선택 확인

`$ARGUMENTS`가 `opus` 또는 `fable`이면 그대로 사용합니다. 비어 있거나 다른 값이면
사용자에게 아래 기준을 짧게 보여주고 물어보세요 (직접 코드를 실행하기 전에 반드시
확인):

| | Opus 4.8 | Fable 5 |
|---|---|---|
| 성격 | 상시 사용 가능한 최상위 안정 모델 | Opus보다 위, 초장기/모호한 문제 전용 |
| 비용 | 기준 | Opus의 약 2배, 사용량 소진도 약 2배 빠름 |
| Thinking | effort로 조절 가능 | 항상 켜짐 (끌 수 없음) |
| 적합한 경우 | 대부분의 코딩/디버깅/설계 작업 | 여러 시간 걸리는 자율 작업, 결과 목표만 던져놓고
  스스로 조사·계획·검증까지 맡기고 싶을 때 |
| 주의 | - | 보안/생물학 관련 내용이 감지되면 세션이 자동으로 Opus로
  폴백될 수 있음 (`/model fable`로 복귀) |

## 2. 프로젝트 기본값 저장

선택이 정해지면 프로젝트 루트의 `.claude/settings.local.json`을 읽고(없으면 새로
만들고) 다음처럼 `model`과 `effortLevel`을 갱신하세요. 기존 내용은 보존하고 이
두 필드만 병합합니다:

- `opus` 선택 시: `"model": "opus"`, `"effortLevel": "xhigh"`
- `fable` 선택 시: `"model": "fable"`, `"effortLevel": "high"`

(`effortLevel`은 설정 파일에는 `low`/`medium`/`high`/`xhigh`만 저장할 수 있고
`max`는 세션 한정이라 저장할 수 없습니다. 정말 어려운 작업 하나에 한해서는
저장된 값과 별개로 그때그때 `/effort max`를 직접 입력하라고 안내하세요.)

## 3. 사용자에게 안내

저장 후 다음을 짧게 안내하세요:
- 파일에 저장한 내용은 **다음 세션부터** 자동 적용된다는 것
- **지금 세션**에 바로 적용하려면 직접 `/model opus` 또는 `/model fable`을
  입력해야 한다는 것 (플러그인이 대신 실행할 수 없음)
- 가장 어려운 작업 한 건에 한해 `/effort max`를 추가로 쓸 수 있다는 것
